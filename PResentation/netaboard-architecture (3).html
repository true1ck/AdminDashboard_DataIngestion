◈ RESILIENCE LAYER — 9 GAPS IDENTIFIED & FIXED
Critical
Single Point of Failure — API
Fix: HAProxy + 2× FastAPI instances + Keepalived VIP. Auto-failover <5s. Nginx SSL termination on SRV-01.
Critical
No Secrets Management
Fix: HashiCorp Vault on SRV-01. All API keys / passwords fetched at runtime. Never stored on disk. Auto-rotation.
Critical
No Dead Letter Queue (DLQ)
Fix: DLQ topic per pipeline stage (enrichment.dlq, embed.dlq). Failed messages after 3 retries → DLQ → ops alert → manual review.
High
No Circuit Breaker
Fix: Tenacity circuit breaker on all gRPC calls to GPU servers. Opens after 5 fails → falls back to rule-based summariser.
High
No Service Discovery
Fix: Consul on all servers. Services register on start. gRPC resolves via DNS (llm-a.service.consul:50051). Health-aware routing.
High
No Backup / DR Strategy
Fix: pg_dump → MinIO daily. Qdrant snapshots weekly. ES ILM snapshots. rclone off-site nightly. RTO <4h, RPO <24h.
Optimise
Late Deduplication
Fix: Redis Bloom Filter at connector level. Content hash checked before Kafka publish. 99.9% of duplicates blocked before pipeline entry.
Optimise
No Data Validation Gate
Fix: Pydantic v2 on every Kafka message deserialization. Avro schema via Schema Registry. Invalid records → validation.errors topic.
Optimise
Unsafe Model Deployment
Fix: MLflow Model Registry: Staging → Production promotion. Canary: 10% traffic to new model. Rollback in <60s if quality drops.
